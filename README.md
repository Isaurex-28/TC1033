# TC1033
# Objetivo
Este proyecto busca resolver la problemática en la que se encontraría una cafetería con problemas administrativos
# Contexto
La cafetería tiene un grave problema de inventario pues los insumos caducan antes de ser utilizados, los desechables se acaban durante la operación
y se busca tener un registro de los clientes frecuentes y sus gustos.
# Funcionamiento
El programa divide la base de datos en 3 clases con atributos y métodos propios (Insumo,Util y Cliente) los objetos pertenecientes a estas clases podrán
ser utilizados y modificados para el correcto control de la cafetería.
